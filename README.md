## 📌 Project Overview

**AI-Powered SDN Load Balancing Dashboard** is a real-time visualization layer for an SDN load balancing project. It serves a modern **Tailwind CSS frontend** through **Flask** and refreshes metrics every second from JSON files generated by either:

- a **Windows-native SDN simulator**, or
- a **Linux Ryu / Mininet / Open vSwitch controller setup**, or
- a safe **Flask-generated fallback demo mode** when no live metrics are available.

> ⚠️ The dashboard itself does **not** start Mininet, Ryu, Open vSwitch, or the Windows simulator.  
> Those processes remain separate and communicate with the dashboard through JSON files.

## ⚙️ Metrics Source Priority

The dashboard reads metrics in this order:

| Priority | Source File | Purpose |
|---:|---|---|
| 1 | `data/live/windows_live_metrics.json` | Windows-native simulator metrics |
| 2 | `data/live/live_metrics.json` | Linux Ryu/Mininet controller metrics |
| 3 | Flask fallback demo data | Animated demo when no live file exists |

If no live metrics are available, the API returns changing demo data with:

```json
"data_source": "demo_fallback"
```

If file-backed metrics are used, the API returns:

```json
"data_source": "live_metrics_file"
```

## 🕹️ Dashboard Controls

The dashboard control buttons update:

```text
data/live/control.json
```

Available actions:

| Button | Backend Endpoint | Action |
|---|---|---|
| RR Mode | `/api/set_mode?mode=rr` | Select Round Robin |
| WRR Mode | `/api/set_mode?mode=wrr` | Select Weighted Round Robin |
| AI / DQN Mode | `/api/set_mode?mode=ai` | Select AI-based routing |
| Normal Traffic | `/api/start_traffic?type=normal` | Start normal traffic |
| High Load | `/api/start_traffic?type=high_load` | Start high-load traffic |
| Burst Traffic | `/api/start_traffic?type=burst` | Start burst traffic |
| Random Traffic | `/api/start_traffic?type=random` | Start random traffic |
| Stop Traffic | `/api/stop_traffic` | Stop traffic generation |

> The dashboard writes commands only.  
> A simulator or controller must watch `control.json` and perform the actual network/simulation action.

## 🪟 Connect a Windows Simulator

Write live metrics to:

```text
data/live/windows_live_metrics.json
```

The dashboard automatically prefers this file over Linux metrics.

Recommended write pattern:

1. Generate metrics in memory.
2. Write to a temporary file.
3. Atomically replace `windows_live_metrics.json`.

This avoids Flask reading a partially written JSON file.

## 🐧 Connect Linux Ryu / Mininet Mode

Write controller metrics to:

The Linux controller remains separate from the Flask dashboard process.

## 🧠 Algorithm Explanation

### 🔁 Round Robin

Round Robin distributes flows sequentially across available paths.

```text
Path 1 → Path 2 → Path 3 → Path 1 → ...
```

Best for:
- Equal path distribution
- Simple baseline comparison
- Low-complexity routing

---

### ⚖️ Weighted Round Robin

Weighted Round Robin prefers better paths more frequently.

Example:

```text
Path 1 weight = 3
Path 2 weight = 2
Path 3 weight = 1
```

Best for:
- Capacity-aware load sharing
- More realistic routing
- Reducing congestion on weaker paths

---

### 🧠 AI / DQN Mode

AI/DQN mode selects paths based on live network conditions such as:

- Current utilization
- Latency
- Packet loss
- Congestion
- Flow count
- Historical reward

Best for:
- Adaptive routing
- Congestion avoidance
- Intelligent traffic engineering

## 🧯 Troubleshooting

<details>
<summary><b>Dashboard opens but values do not change</b></summary>

Check:

```bash
curl http://127.0.0.1:5000/api/metrics
```

If `"data_source": "demo_fallback"`, your simulator/controller is not writing live metrics yet.

</details>

<details>
<summary><b>Browser shows blank page</b></summary>

Check Flask logs and confirm:

```text
dashboard/templates/index.html
```

exists.

Then restart:

```bash
python dashboard/app.py
```

</details>

<details>
<summary><b>Control buttons work but network does not change</b></summary>

The buttons only update:

```text
data/live/control.json
```

Your simulator/controller must watch this file and apply the command.

</details>

<details>
<summary><b>Remote Tailwind/font/icons are not loading</b></summary>

The UI uses remote Tailwind runtime, Google Fonts, and Material Icons. Metric updates still work, but styling/icons may look different without internet.

</details>

## ⚠️ Limitations

- Demo metrics are presentation fallback data only and are labeled as `demo_fallback`.
- Dashboard command endpoints persist commands but do not execute network actions directly.
- Mininet, Ryu, Open vSwitch, and Windows simulator processes must run separately.
- Remote Tailwind, Google Fonts, Material icons, and topology images require internet for full visual styling.
- This dashboard is a visualization and control layer, not a replacement for the SDN controller.
